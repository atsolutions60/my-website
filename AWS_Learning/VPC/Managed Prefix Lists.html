Managed Prefix Lists

Introduction

AspectDescription
Definition A managed prefix list is a set of one or more CIDR blocks. You can create a prefix list from the IP addresses that you frequently use, and reference them as a set in security group rules and routes instead of referencing them individually.
Usage You can reference a prefix list with:
  • VPC security groups
  • Subnet route tables
  • Transit gateway route tables
  • AWS Network Firewall rule groups
  • Amazon Managed Grafana network access control

Types of Prefix Lists

TypeDescription
Customer-managed prefix lists
  • Sets of IP address ranges that you define and manage.
  • You can share your prefix list with other AWS accounts.
  • Prefix list supports a single type of IP addressing only (IPv4 or IPv6).
  • A prefix list applies only to the Region where you created it.
AWS-managed prefix lists
  • Sets of IP address ranges for AWS services.
  • You cannot create, modify, share, or delete an AWS-managed prefix list.
