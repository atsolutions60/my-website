AWS VPC Endpoints

Introduction

Aspect Description
Definition A VPC endpoint enables you to privately connect your VPC to supported AWS services and VPC endpoint services powered by PrivateLink without requiring:
Without Endpoints
  • Internet gateway
  • NAT device
  • VPN connection
  • AWS Direct Connect connection
Nature Endpoints are virtual devices. They are horizontally scaled, redundant, and highly available VPC components that allow communication between instances in your VPC and services without imposing availability risks or bandwidth constraints.

Before & After EndPoint

AWS PrivateLink Introduction

Feature Description
Security AWS PrivateLink simplifies the security of data shared with cloud-based applications by eliminating the exposure of data to the public Internet.
Connectivity AWS PrivateLink provides private connectivity between VPCs, AWS services, and on-premises applications, securely on the Amazon network.
Simplification AWS PrivateLink makes it easy to connect services across different accounts and VPCs to significantly simplify the network architecture.

Types of VPC Endpoints

Type Description
Gateway Endpoints A gateway endpoint is for supported AWS services only. You specify a gateway endpoint as a route table target for traffic destined to:
  • Amazon S3
  • DynamoDB
Interface Endpoints An interface endpoint is an elastic network interface with a private IP address from the IP range of your subnet. It serves as an entry point for traffic destined to a supported AWS service or a VPC endpoint service. Interface endpoints are powered by AWS PrivateLink.
Gateway Load Balancer Endpoints A Gateway Load Balancer endpoint is an elastic network interface with a private IP address from the IP range of your subnet. These endpoints are powered by AWS PrivateLink and serve as an entry point to intercept traffic and route it to a service configured using Gateway Load Balancers.
