AWS DHCP Option Set

Introduction

Aspect Description
Definition A DHCP Option Set in AWS provides custom DHCP (Dynamic Host Configuration Protocol) options for your VPC, such as domain name, DNS servers, and NTP servers.
Purpose It allows you to control how instances launched in a VPC receive network configuration settings dynamically.

Key Components

Component Description
Domain Name Specifies a custom domain name for instances in the VPC (e.g., example.com).
Domain Name Servers List of DNS servers (e.g., AmazonProvidedDNS or custom IPs).
NTP Servers Specifies one or more Network Time Protocol (NTP) servers for time synchronization.
NetBIOS Name Servers List of servers for NetBIOS name resolution (for legacy Windows workloads).
NetBIOS Node Type Defines how NetBIOS clients resolve names (e.g., broadcast or WINS).

Use Cases

Scenario Description
Custom DNS Direct instances to use a custom DNS server (e.g., an on-premises DNS resolver).
Time Synchronization Provide your own NTP servers for better control of time-sensitive applications.
Hybrid Environments Integrate with existing corporate DNS and name resolution infrastructure when extending workloads into AWS.

Considerations

Consideration Description
Default Option Set Every VPC automatically gets a default DHCP option set with AmazonProvidedDNS.
One per VPC You can only associate one DHCP option set with a VPC at a time, but you can replace it anytime.
Global Scope DHCP option sets are scoped to a Region, not a single VPC.
