RDS Maintenance: Minor Updates only

Introduction

Description
The Maintenance Window in Amazon RDS is a set time for AWS to perform maintenance on your database.
During this time, AWS may apply security patches, software updates, and minor version upgrades.
This keeps your RDS instance secure and running smoothly.

Key Points

Topic Details
Why It’s Important Maintenance helps fix security issues, improve performance, and ensure the database is up-to-date.
Choosing a Maintenance Window • You can pick a day, time, and duration (up to 8 hours) for maintenance.
• Best to select low-traffic hours to avoid disruption.
Automatic Minor Upgrades • AWS can automatically apply minor updates during the window.
• Example: 5.6.1 → 5.6.2 is a minor update.
• Major updates (e.g., 5.6 → 5.7) are not automatic.
Major Version Upgrades • These are larger updates that may change features or configurations.
• Must be triggered manually for compatibility testing.
Tasks Done During Maintenance • Security patches
• Software updates
• Hardware maintenance
Adjusting the Window You can modify the maintenance window at any time in the RDS console.
Notifications AWS sends notifications for upcoming maintenance so you can prepare.

Maintenance Considerations

RDS Deployment Type Behavior During Maintenance
Single-AZ • Maintenance causes downtime.
• Best for non-critical workloads.
• Select low-traffic maintenance hours.
Multi-AZ • Updates applied first to standby instance.
• RDS then performs a quick failover.
• Downtime is minimized.
Multi-AZ DB Cluster • Updates are applied gradually across nodes.
• No downtime or service interruption.
Deletion Protection • Prevents accidental deletion of your RDS instance.
• Must be turned off manually before deleting the instance.
• Recommended for production environments.
