Amazon RDS Proxy

Introduction

Description
Amazon RDS Proxy is a service that acts like a helper between your application and your database.
It manages database connections, makes your application faster, handles more users at the same time, and keeps your database secure and reliable.
Itâ€™s especially useful for apps like AWS Lambda, which can have many users connecting to the database at once.

Without RDS Proxy vs With RDS Proxy

Parameter
Without RDS Proxy
With RDS Proxy
Connections AWS Lambda, ECS, and EKS are all connecting directly to Amazon RDS. Each service manages its own database connections. Amazon RDS Proxy sits between your application and the database. Applications connect to the proxy endpoint instead of the database.
Key Issue / Benefit Direct connections can lead to performance bottlenecks, poor performance, and difficulty handling failover. The proxy manages connections efficiently and improves application performance by pooling and reusing connections.
Connection Pooling No connection pooling. Each request may create a new connection, increasing load. Proxy maintains a pool of pre-established connections and reuses them, reducing resource usage.
Failover Handling Failover handling is inefficient, causing delays or failures. Automatically switches connections to the standby database (Multi-AZ) during failover.
Security Application directly handles database credentials. Uses AWS Secrets Manager for secure authentication. Applications do not handle DB credentials directly.
