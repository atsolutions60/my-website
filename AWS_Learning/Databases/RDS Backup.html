RDS Automated Backups

Introduction

Description
Automated backups enable RDS to take a daily, incremental snapshot of your database.
AWS manages these snapshots for you, ensuring they are protected and available for recovery.
The first backup is a full snapshot, and subsequent backups are incremental, meaning they only store changes made since the previous backup.
This approach saves storage space and reduces backup time.

Location & Storage

Description
Automated backups are stored in Amazon S3 within the same AWS Region as your RDS instance.
AWS manages these backups transparently, and they aren’t directly accessible to users.
Instead, they are controlled through the RDS console, CLI, or API for ease of use and security.

Backup Retention Period

Description
The backup retention period determines how long your backups are stored.
You can set this retention period from 1 to 35 days (the default is usually 7 days).
If set to 0 days, automated backups are disabled.
AWS will automatically delete any backups older than the retention period, keeping only the most recent data available for restoration.

Scheduled Backup Window

Description
Automated backups are scheduled during a specified daily backup window.
You can define this window, or AWS will automatically assign one.
During this window, RDS takes a full snapshot if needed and performs any incremental backups.
The backup window is especially useful for minimizing performance impacts, as it lets you specify a low-traffic time when the database may experience minimal use.

Backup Replication

Description
By default, automated backups remain in the original AWS Region.
You can replicate automated backups to another AWS Region to help with disaster recovery.
Snapshots and transaction logs are replicated immediately after they are available in the source.
Automated backup replication is not supported for Multi-AZ DB clusters.
There are no additional data transfer charges for backups stored within the same AWS Region, but charges apply for snapshot copies across regions.

Costs Associated with Automated Backups

Description
RDS provides backup storage up to the size of the DB instance’s storage for free.
If your DB instance has 200 GB of storage, you get 200 GB of backup storage for free.
Additional storage beyond this free allocation incurs a charge, which includes both automated backups and manual snapshots.

Restoration Options

Restoration TypeDescription
Restoring to Latest Backup You can quickly restore your database to the latest backup available within your retention period. This recovery method is fast and typically suitable for instances where a recent copy of the database is needed.
Point-in-Time Recovery (PITR) With automated backups enabled, RDS allows you to restore your database to any specific point in time within the backup retention period. PITR is useful for recovery from unexpected data changes or corruption. For example, if accidental deletions or modifications occur, you can restore the database to just before the event happened.

Manual Snapshots for Longer Retention

Description
While automated backups have a limited retention period, you can create manual snapshots anytime.
These snapshots are user-initiated and are retained until explicitly deleted, making them useful for long-term backups or before significant changes to the database.
