Choosing the Availability Option

Category Explanation
Introduction AWS RDS offers three availability models:
• Single Host
• Multi-AZ
• Multi-AZ Cluster
RPO (Recovery Point Objective) • Measures acceptable data loss.
• Backup frequency defines RPO.
RTO (Recovery Time Objective) • Maximum allowable downtime.
• Single Host → High RTO
• Multi-AZ → Low RTO
• Multi-AZ Cluster → Very low RTO
Availability Comparison — Single Host • RPO: High (backup-based)
