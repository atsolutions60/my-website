RDS Encryption

Introduction

Description
Encryption in Amazon RDS (Relational Database Service) provides enhanced security by protecting data at rest and in transit.
RDS encryption uses the AWS Key Management Service (KMS) to encrypt your database, backups, snapshots, and read replicas.

Purpose of RDS Encryption

Description
Protect sensitive data in the database by making it inaccessible without proper authorization.
Ensure compliance with industry standards, regulatory requirements, and security best practices.
Secure backups and snapshots associated with the encrypted instance.
Encrypt data in transit to prevent unauthorized access while data moves within and outside the AWS network.

Setting Up Encryption

Description
RDS encryption must be enabled when creating a new DB instance.
It cannot be added to an existing, non-encrypted DB instance.

Steps to Enable Encryption

Step Details
Create Snapshot Create an unencrypted snapshot of your existing DB instance.
Copy Snapshot Copy the snapshot with encryption enabled.
Restore Restore the encrypted snapshot to a new encrypted DB instance.

KMS Key Options

Key Type Description
Default AWS KMS Key AWS automatically manages a default key, which is convenient if you want encryption without managing your own keys.
Customer-Managed Key You can create and manage your own KMS keys, defining specific permissions and access policies. Provides more control but requires additional management.

Data in Transit Encryption

Description
In addition to encryption at rest, RDS also supports encryption for data in transit.
This feature uses SSL/TLS encryption to secure data transmitted between your applications and RDS instances.
RDS provides a certificate for SSL/TLS connections.
You configure your application to use SSL/TLS, ensuring that data sent between the app and RDS is encrypted.
