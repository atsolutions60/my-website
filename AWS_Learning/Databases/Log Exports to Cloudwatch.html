RDS Log Exports

Introduction of Logs

Description
Logs are records of various database events and activities.
Logs capture information on queries, errors, connections, and access activities, helping track what’s happening inside the database.

Types of Logs

Log Type Description
Audit Logs (for security) Track user activities for compliance and security purposes.
Error Log Record errors that help in diagnosing issues.
General Log Capture information about connections and commands.
Slow Query Log Identify slow-running queries for performance tuning.

Log Exports in RDS

Description
Log Exports in RDS allow you to centralize and store logs outside of the database, making it easier to retain, monitor, and analyze them over time.

Destinations & IAM Role

Topic Details
Default Destination When you configure Log Exports in Amazon RDS, the destination for the logs is automatically set to Amazon CloudWatch Logs.
Destination Options There is no option in RDS to choose a custom destination; by default, all exported logs always go to CloudWatch Logs.
Exporting to S3 If long-term archival is needed, logs can be exported from CloudWatch to Amazon S3.
IAM Role An IAM role is required to allow Amazon RDS to publish log data into CloudWatch Logs.
