Parameter Group vs Option Group

Parameter Group Option Group
Purpose
Controls internal database engine settings like memory management, connection limits, and performance tuning. 		Purpose
Adds extra features or extensions to the RDS instance beyond core DB engine capabilities.
Examples of Parameters
• max_connections — Controls how many clients can connect
• query_cache_size — Speeds performance by caching query results 		Examples of Options
• Oracle OEM Plugin — Web-based DB monitoring
• SQL Server TDE — Encrypts data at rest
• MySQL MEMCACHED — In-memory caching extension
Use Case
Modify DB engine behavior such as tuning performance or increasing connection capacity. 		Use Case
Enable specialized add-ons or functionality not available through normal DB settings.

Example Scenario

Suppose you have a MySQL RDS instance:

• Use a Parameter Group to increase max_connections during heavy traffic.

• Use an Option Group to enable MEMCACHED for advanced caching capabilities.

Summary:

• Parameter Groups: Control internal DB behavior
• Option Groups: Enable extra DB features and extensions
