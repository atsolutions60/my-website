CloudFront — Setting Options

Supported HTTP versions

Description
• When configuring a CloudFront distribution, you have the option to specify which versions of the HTTP protocol you want to support
• HTTP/1.0 and HTTP/1.1 are supported by default.
• The options typically include HTTP/2 and HTTP/3, both of which offer significant performance and security benefits over the older HTTP/1.1 protocol
• CloudFront will automatically use the best version supported by the viewer's browser or client
• HTTP/2

• HTTP/2 is an upgrade over HTTP/1.1 and is widely supported by modern browsers • It improves speed and efficiency by enabling multiplexing and header compression.
• HTTP/3

• Takes these improvements further by using QUIC and UDP • Offers lower latency and improved performance, especially on unstable networks.

Default root object

Description
• Allows you to specify a default file (usually an HTML file) that CloudFront will serve when a user accesses the root URL of your distribution
• Example

• When users visit your CloudFront distribution without specifying a file (https://www.example.com/ instead of https://www.example.com/index.html)

Standard logging

Description
• Standard logging in CloudFront enables you to log information about every request made to your distribution.
• These logs are stored in an Amazon S3 bucket that you specify
• Logs can include:

• Date and Time of the request • Client IP Address • Request URL • Status Code • Bytes Transferred

Cookie logging

Description
• This feature allows you to capture and log the name and value of cookies included in requests to your CloudFront distribution.
• The logged cookie data can be useful for understanding user behavior, session management, and customizing content delivery based on user-specific data.

IPv6

Description
• The IPv6 setting in Amazon CloudFront allows you to enable or disable support for IPv6 addresses in your distribution.
• Use Case

• Future-Proofing • Global Reach • Network Performance
