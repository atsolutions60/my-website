CloudFront Origin Group

Introduction

Description
  • CloudFront Origin Group
  • A CloudFront Origin Group is a collection of multiple origins
  • It provides reliability for your content delivery
Multiple Origins
  • You can add multiple origins to the group
  • These origins can be defined as:
    • Primary Origin
    • Secondary Origin
Primary Origin
  • The primary origin is the main source for your content
  • CloudFront sends all requests to the primary origin first.
Secondary Origin
  • The secondary origin acts as a backup
  • If the primary origin fails, CloudFront switches to the secondary origin
Automatic Failover
  • CloudFront automatically checks the health of both origins
  • It automatically switches to the secondary origin if the primary is unavailable

Geographical Failover with Load Balancers

Description
Scenario Explanation
  • In this advanced scenario, the goal is to ensure your website remains highly available and resilient, even in the event of a regional failure.
  • This setup uses CloudFront with a geographically dispersed failover strategy.
Why We Are Creating This Setup ?
  • Global Reliability
    • If the primary load balancer in the India region goes down, we want users to still access the website.
    • Failover to N. Virginia ensures site availability even if a whole region is down.
  • Business Continuity on a Global Scale
    • For global businesses, redundancy across regions is essential.
    • This scenario ensures operations continue uninterrupted.
Hands-On Steps
  • Step-1 Create EC2 Instances in both regions and configure as web servers
  • Step-2
    • Set up a load balancer in the India region
    • Set up a load balancer in the N. Virginia region
  • Step-3 Create a CloudFront Distribution
  • Step-4 Setup Origin Group
  • Step-5 Test the Setup
