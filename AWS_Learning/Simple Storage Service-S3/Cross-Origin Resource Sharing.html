Cross-Origin Resource Sharing (CORS)

Enable secure communication between resources from different domains using AWS S3.

AspectDescription
DefinitionCORS (Cross-Origin Resource Sharing) allows web applications hosted on one domain to request resources from another domain securely.
Browser RestrictionBy default, web browsers block cross-origin requests for security reasons; CORS provides a mechanism to allow safe access when configured.
PurposeTells browsers that it‚Äôs safe to fetch resources from another domain, such as another S3 bucket or API endpoint.

Why Use CORS with Amazon S3

ScenarioExplanation
Multi-Bucket WebsitesIf your static website is hosted in one S3 bucket but needs to access images or APIs from another S3 bucket, CORS enables this safely.
SecurityWithout CORS, browsers block cross-domain requests. With proper setup, the browser allows controlled access across origins.
ConfigurationYou define a CORS configuration in the source bucket specifying which domains, methods, and headers are allowed.
ResultOnce enabled, your web app can securely load resources such as images, JSON, or files from another S3 bucket or AWS service.

Sample CORS Configuration Rule

{
  "CORSRules": [
    {
      "AllowedHeaders": ["*"],
      "AllowedMethods": ["GET", "POST", "PUT"],
      "AllowedOrigins": ["*"],
      "ExposeHeaders": [],
      "MaxAgeSeconds": 3000
    }
  ]
}
