Lifecycle Rule Options Explained

These lifecycle configuration options determine how S3 manages current and noncurrent object versions automatically.

Rule Option Description Example Use Case
Move current versions of objects between storage classes Transitions current (latest) object versions from one S3 storage class to another to reduce cost. Example: After 30 days of inactivity, move objects from S3 Standard to S3 Standard-IA.
Move noncurrent versions of objects between storage classes Transitions older versions of objects (from versioned buckets) to cheaper storage classes. Example: Move previous versions of files older than 60 days from Standard to Glacier Flexible Retrieval.
Expire current versions of objects Deletes the latest version of an object automatically after a defined period. Example: Delete log files older than 365 days to save storage.
Permanently delete noncurrent versions of objects Removes outdated object versions permanently to minimize storage usage in versioned buckets. Example: Automatically delete noncurrent object versions after 180 days.
Delete expired object delete markers or incomplete multipart uploads Removes leftover delete markers and unfinished uploads to keep the bucket clean and avoid costs. Example: Automatically delete incomplete multipart uploads older than 7 days.
