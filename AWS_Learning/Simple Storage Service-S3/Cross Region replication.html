Amazon S3 Cross-Region Replication (CRR)

Automatically replicate S3 objects across AWS Regions for durability, compliance, and low-latency access.

AspectDescription
DefinitionCRR automatically replicates objects from a source S3 bucket to a destination bucket in another AWS Region.
PurposeImprove data durability, enable global access, meet compliance requirements, and support disaster recovery.
Replication DirectionOne-way replication (from source to destination). It‚Äôs not bi-directional by default.
Replication ScopeEntire bucket or specific objects based on prefix or tag filters.
Use Case ExampleReplicate customer data uploaded in us-east-1 to ap-south-1 for compliance or global availability.

Cross-Region Replication (CRR) vs Same-Region Replication (SRR)

Feature Cross-Region Replication (CRR) Same-Region Replication (SRR)
Replication LocationReplicates data between different AWS Regions.Replicates data within the same AWS Region.
Primary UseDisaster recovery, latency optimization, compliance, and global distribution.Backup, log aggregation, and access segregation within the same region.
Data Transfer CostCross-region transfer fees apply.No cross-region transfer cost (same-region only storage cost).
LatencyHigher, due to inter-region replication.Lower, since replication occurs within the same region.
Compliance & DRMeets compliance standards requiring geographic separation (e.g., GDPR).Does not satisfy cross-regional compliance requirements.
Common Use Cases
  • Global data distribution
  • Geo redundancy
  • Compliance archiving
  • Data protection within same region
  • Access isolation between environments
  • Audit and log replication

Key Requirements for CRR Setup

RequirementDetails
VersioningMust be enabled on both source and destination buckets.
PermissionsSource bucket owner must have permission to replicate to the destination bucket.
IAM RoleA replication IAM role must be defined with s3:ReplicateObject and related permissions.
EncryptionSupports replication for both SSE-S3 and SSE-KMS encrypted objects (requires KMS permissions).
Destination Bucket OwnershipDestination bucket can be in the same or different AWS account.
Replication Time Control (RTC)Optional feature to ensure replication completes within 15 minutes (for premium workloads).

Benefits of Using Cross-Region Replication

BenefitExplanation
Disaster RecoveryAutomatic duplication of data to a remote region ensures business continuity.
Data SovereigntyMeet regulations requiring data storage in specific regions.
Improved AccessServe users from geographically closer regions with replicated copies.
Backup & ArchiveActs as an automatic backup for long-term data durability.
Multi-Account ArchitectureSupports replication between AWS accounts for separation of duties or organizational boundaries.
