Amazon S3 Object Lock

Ensures data immutability by preventing object deletion or modification for a fixed time or indefinitely.

Feature Description
Purpose Prevents deletion or modification of an S3 object or its version for a defined retention period or indefinitely.
Benefits Protects data from accidental or malicious deletion and ensures compliance with regulatory retention requirements.
Use Cases
  • Regulatory compliance requiring immutable storage
  • Legal investigations where data must be preserved
  • Protection against human or system errors

Terminology and Modes

Key terms and configuration modes used in S3 Object Lock.

Term Explanation Example / Notes
Retention Mode Defines how strictly an object version is protected for a specific duration. Two modes: Governance and Compliance.
Governance Mode Allows only users with special permissions to delete or modify objects before retention ends. Requires permissions like s3:BypassGovernanceRetention and s3:PutObjectRetention.
Compliance Mode Strict mode preventing modification or deletion by any user, including the root account, until retention expires. Used for highly regulated workloads (e.g., financial or healthcare data).
Retention Period Specifies how long the object version remains protected from deletion or overwrite. Can be defined in days or years at object or bucket level.
Legal Hold Provides indefinite protection with no predefined expiration. Useful during legal investigations where retention length is unknown.

Enabling S3 Object Lock

Steps and requirements for configuring Object Lock on an S3 bucket.

Requirement Details
Versioning Bucket versioning must be enabled before you can enable Object Lock.
Permissions The following IAM permissions are required:
  • s3:PutObjectRetention
  • s3:GetObjectRetention
  • s3:PutObjectLegalHold
Enable Object Lock
  • Must be activated during bucket creation.
  • Once enabled, cannot be disabled.
  • Allows applying retention policies and legal holds.

Default Retention Configuration

Defines whether all new objects in a bucket automatically receive a retention policy.

Setting When to Use Advantages
Disabled
  • When you want full control over which objects are protected.
  • When object lifespans vary within the same bucket.
 Provides flexibility and easier manual management.
Enabled
  • When consistent protection is required for all objects.
  • When compliance regulations demand immutability.
  • To reduce manual configuration and errors.
 Applies automatic protection to all new objects.
