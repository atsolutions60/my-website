Amazon Route 53 Overview

What is Route 53?

AspectDescription
Definition AWS’s highly available DNS service. Used to route users to websites, apps, and services using domain names.

Common Record Types

Record TypeUse Case
AMaps domain to IPv4 address
AAAAMaps domain to IPv6 address
CNAMEAlias for another domain name
MXMail server
TXTText record (e.g., SPF, DKIM validation)
NSPoints to name servers of hosted zone
AliasAWS-specific record pointing to AWS resources (like ELB, S3, CloudFront)

Routing Policies

PolicyDescription
Simple RoutingOne record → one value (A or CNAME). No health checks or failover.
Latency RoutingRoutes traffic to region with lowest latency. Example: Users in India → Indian server.
Geolocation RoutingRoutes traffic based on user’s geographic location.
Geoproximity RoutingRequires Route 53 + Traffic Flow. Routes based on location + bias setting. You can expand/contract regions manually.
Failover RoutingActive-passive configuration. Routes traffic to primary (healthy). If primary fails, traffic goes to secondary.
Weighted RoutingAssign weights (e.g., 70/30 split). Route traffic proportionally.
Multivalue Answer RoutingLike simple routing but returns multiple IPs. Basic load balancing. Supports health checks.

Health Checks

FeatureDescription
MonitoringMonitors endpoint health (HTTP, HTTPS, TCP).
IntegrationOptional integration with CloudWatch alarms.
RoutingIf health check = up → traffic routed. If health check fails → switch/remove resource from rotation.

SAA-C03 Exam Tips

TipDetails
Geolocation vs GeoproximityGeolocation = by country. Geoproximity = by location with bias control.
Disaster RecoveryUse failover for DR and high availability.
User ExperienceUse latency-based routing for best UX.
CNAME LimitationCNAME cannot be used at root domain (use Alias instead).
Alias RecordsAWS-only, free, and support root domain.
