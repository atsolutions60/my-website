Cross-Zone Load Balancing

Overview

FeatureDescription
Purpose Cross-Zone Load Balancing determines whether each load balancer node distributes traffic only to targets in its own Availability Zone or across targets in all enabled Availability Zones.

Behavior Comparison

StateBehavior
When Disabled Each load balancer node distributes traffic across the registered targets in its Availability Zone only.
When Enabled Each load balancer node distributes traffic across the registered targets in all enabled Availability Zones.

Considerations

ServiceDetails
Application Load Balancer (ALB) Cross-Zone Load Balancing is always enabled and cannot be disabled.
Network Load Balancer (NLB) Cross-Zone Load Balancing is disabled by default. You can enable or disable it at any time after creation using the AWS Console, CLI, or API.

Key Points & Exam Tips

TopicExplanation
Traffic Distribution When enabled, traffic is evenly spread across all registered targets in all Availability Zones, improving utilization and resilience.
Use Case Ideal when target counts differ per AZ, ensuring fair distribution across all healthy targets.
Latency Impact May slightly affect latency due to cross-zone routing but provides higher availability overall.
Exam Tip Remember that ALB always has Cross-Zone enabled, while NLB/GWLB have it disabled by default.
