AWS Auto Scaling

Introduction

AspectDetails
PurposeEnsures the correct number of Amazon EC2 instances are available to handle the load for applications.
Minimum InstancesThe group never goes below the specified minimum size.
Maximum InstancesThe group never exceeds the specified maximum size.
Desired CapacityEnsures the group maintains the defined number of instances at all times.

Benefits

BenefitExplanation
Fault ToleranceDetects unhealthy instances, terminates them, and launches replacements. Supports multi-AZ resilience.
High AvailabilityEnsures applications always have sufficient capacity to handle demand.

Scaling Terminology

TypeDescription
Vertical Scaling
  • Scale Up: Add CPU or memory to one node.
  • Scale Down: Remove resources from that node.
Horizontal Scaling
  • Scale Out: Add more nodes or EC2 instances.
  • Scale In: Remove nodes.
Auto ScalingAutomatically performs horizontal scaling.

Auto Scaling Components

ComponentDescription
Auto Scaling GroupsOrganize EC2 instances into logical groups for scaling and management.
Launch Templates / Configurations
  • Launch Configurations define AMI ID, type, key pair, and security groups.
  • Launch Templates Advantages:
    • Versioning support.
    • Access to new features (gp3/io2, EBS tagging).
    • Supports mixed instance types and Spot + On-Demand setups.

Scaling Options

TypeDescriptionUse Case
Manual ScalingManual change of min, max, or desired capacity.Special events (e.g., new app release).
Scheduled ScalingTriggered by date/time schedules.Predictable workloads (month-end).
Dynamic ScalingBased on metrics (e.g., CPU > 90% for 15 mins).Unpredictable loads.

Predictive Scaling

AspectDetails
DefinitionUses ML models to forecast demand using 3+ weeks of data, scaling proactively.
ModesOn: Auto adjust.
Off: Only forecasts.
MetricsCPU, network I/O, or request count.
Target UtilizationDesired usage (e.g., 50% CPU).
Pre-LaunchLaunch instances before forecast time (e.g., 5 mins early).
BufferAdds extra capacity (e.g., +20%).

Instance Maintenance Policy

ModeDescription
Terminate & LaunchRemoves unhealthy instances immediately and replaces them.
Launch Before TerminateStarts new instances first to minimize downtime.
Custom BehaviorAdvanced fine-tuning using health checks and policies.

Termination Policy

PolicyDescription
DefaultBalances across AZs, respects protection, prefers oldest config, and optimizes by billing hour.
Built-inOldestTemplate, ClosestToNextHour, NewestInstance, OldestInstance.
CustomLambda-based smart termination with graceful shutdown, data backup, and flexible logic.
