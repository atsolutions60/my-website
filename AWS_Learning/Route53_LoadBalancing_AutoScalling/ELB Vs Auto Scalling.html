ELB vs Auto Scaling

Auto Scaling

FeatureDescription
Infrastructure Scaling It scales your infrastructure whenever needed.
Automatic Group Adjustment Automatically increases and decreases the size of Auto Scaling group (mainly EC2 instances).
Dynamic Load Handling Adds new instances when load reaches a certain level and spins down instances when no longer needed.

Elastic Load Balancer (ELB)

FeatureDescription
Traffic Distribution Automatically routes incoming application traffic across dynamically changing number of multiple EC2 instances.
Single Entry Point Acts as a single point of contact for all incoming traffic to the instances in your Auto Scaling group.
Health Monitoring Monitors the health of registered instances so the load balancer sends requests only to healthy instances.

Auto Scaling vs ELB

AspectAuto ScalingELB
Primary Function Automatically adjusts the number of EC2 instances based on demand. Distributes incoming network traffic evenly across EC2 instances.
Scaling Scales infrastructure up or down as per defined policies. Does not scale instances but manages traffic distribution.
Health Check Uses EC2 status checks to determine instance health. Performs health checks and routes traffic to healthy instances only.
Integration Often works together with ELB for dynamic load handling. Typically paired with Auto Scaling for fault tolerance and performance optimization.
