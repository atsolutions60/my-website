AWS Gateway Load Balancer (GWLB)

Introduction

AspectDescription
Layer AWS Gateway Load Balancer (GWLB) is a Layer 3 (network layer) service.
Purpose Gateway Load Balancer (GWLB) is a service designed to help you deploy, scale, and manage virtual appliances, such
Appliance Examples
  • Firewalls
  • Intrusion Detection System (IDS)
  • Intrusion Prevention System (IPS)
Capabilities It distributes traffic across multiple appliance instances, performs health checks, and supports Equal-Cost Multi-Path routing (ECMP) for high availability and throughput.
Benefit GWLB simplifies the deployment and management of network appliances in AWS, making it easier to enhance security and networking capabilities in the cloud.

Use Case / Traffic Flow

ScenarioDescription
Traffic Flow Use GWLB to steer traffic to a fleet of virtual appliances for inspection or processing, enabling scalable, highly-available inline network services.
Scaling & HA Distributes traffic across multiple appliance instances and leverages health checks and ECMP to achieve high availability and throughput.
Typical Integrations Often used with third-party firewalls, IDS/IPS, and other network security appliances to centralize inspection and enforcement.

Notes & Best Practices

TopicNotes
Deployment Deploy GWLB in front of virtual appliance fleets; configure target appliances to receive and process traffic forwarded by the GWLB.
Health & Routing Use GWLB health checks to ensure traffic is only sent to healthy appliance instances; ECMP support allows multiple equal-cost paths for higher throughput.
Use Cases Centralized inspection, inline traffic processing, scalable firewall/IDS/IPS deployments.
