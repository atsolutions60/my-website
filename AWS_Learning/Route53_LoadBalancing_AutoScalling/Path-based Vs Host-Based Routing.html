Path-based Routing vs Host-based Routing

Routing Criteria

AspectPath-Based RoutingHost-Based Routing
Definition Routes traffic based on the URL path.
Example: example.com/images or example.com/orders		 Routes traffic based on the domain name in the request.
Example: images.example.com or orders.example.com

DNS Configuration

AspectPath-Based RoutingHost-Based Routing
DNS Setup Points to the same ALB — no specific DNS setup for individual paths. Requires DNS records for each domain or subdomain to point to the ALB.

Use Cases

AspectPath-Based RoutingHost-Based Routing
Best Suited For Routing to different applications or services under the same domain. Routing to different applications or services based on different domains or subdomains.

SSL/TLS Certificates

AspectPath-Based RoutingHost-Based Routing
Certificate Management A single SSL/TLS certificate can cover all paths under one domain. May require multiple certificates if different domains are used.

Flexibility

AspectPath-Based RoutingHost-Based Routing
Scope Limited to routing within a single domain. More flexible — supports multiple domains and subdomains.

Complexity

AspectPath-Based RoutingHost-Based Routing
Management Overhead Generally simpler — fewer DNS records to maintain. More complex — requires managing multiple DNS entries and SSL certificates.

SAA-C03 Exam Tips

TopicExplanation
Routing Criteria Path-based routes by URL path, Host-based routes by domain name.
Certificate Management Use one certificate for path-based routing; multiple certificates may be required for host-based routing.
Flexibility Host-based routing is more flexible, while path-based routing is simpler.
DNS Records Path-based needs no separate DNS entries; Host-based requires DNS entries for each domain.
