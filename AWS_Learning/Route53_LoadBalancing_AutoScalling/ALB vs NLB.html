Network Load Balancer (NLB)

Overview

FeatureDescription
Purpose Network Load Balancers are best suited for load balancing of TCP, UDP, and TLS traffic where extreme performance and ultra-low latency are required.
Operation Layer Operates at the connection level (Layer 4) of the OSI model.
Performance Capable of handling millions of requests per second while maintaining ultra-low latency.

Application vs Network Load Balancer

Feature Application Load Balancer (ALB) Network Load Balancer (NLB)
Supported Protocols HTTP, HTTPS, gRPC TCP, UDP, TLS
Load Balancer Type Layer 7 Layer 4
Sticky Sessions (Cookies) Yes — helps route requests from the same client to the same target Yes — supported for certain target groups
Idle Connection Timeout Yes — reuses TCP connections for a short duration Not Applicable (connection-level load balancing)
Path & Host-Based Routing Yes — supports routing based on URL path or hostname No — distributes purely based on network information
WebSockets Support Yes — supports bidirectional communication Yes — supported at transport layer
Static IP Support No — uses DNS names Yes — provides one static IP per AZ

SAA-C03 Exam Tips

TopicExplanation
Protocol Level NLB operates at Layer 4 while ALB operates at Layer 7.
Performance NLB offers ultra-low latency and high throughput — ideal for millions of requests per second.
Use Cases Choose NLB for TCP/UDP traffic or static IP requirements; ALB for advanced routing (path/host-based, HTTP features).
Static IPs Only Network Load Balancer supports static IP addresses.
